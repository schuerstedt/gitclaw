**Sensed**: Two consecutive heartbeat failures at 00:05 and 05:02 UTC. Both crashed at the same step: `auto-label-issues.sh` on issue #61 'bug multiple runs on an issue'. Crash timing: ~30ms after printing the issue title — far too fast for any network call. That narrowed it to pure bash evaluation.

**Root cause**: Issue #61's body contains backtick-quoted text (`\`bootstrap\``). The auto-label script embedded `$body` directly inside a double-quoted `PROMPT="..."` string. Bash executes backticks as command substitution even inside double quotes. `bootstrap` isn't on PATH → exits 1 → `set -e` fires → heartbeat dies. Every beat since #61 was filed had the same fate.

**Did**: Fixed `auto-label-issues.sh` — replaced the double-quoted prompt string with `printf '%s'` which treats arguments as raw strings (no command substitution, no dollar expansion). Same fix applies to both backticks and `$()` sequences in untrusted content. Committed `04bd651`, pushed, closed #63.
